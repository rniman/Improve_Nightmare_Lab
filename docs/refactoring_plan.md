# Refactoring Plan

## 목표
- 현재 아키텍처를 명확히 파악
- 렌더링 및 네트워크 흐름을 문서화
- 기술 부채를 점진적으로 줄이기
- 모듈 단위 리팩토링 시 네이밍 개선
- 시스템을 충분히 이해할 때까지 디렉토리 재구성은 보류
- 구조 정리가 완료된 후에만 성능 최적화 수행

## 우선순위
1. 파이프라인 및 의존성 분석
2. 문서 업데이트
3. 점진적 리팩토링 (작고 국소적인 변경)
4. 디렉토리 재구성 (별도의 명시적 작업으로 분리)
5. 성능 최적화

## 누락된 안전 점검 사항 (추가)
- 리팩토링 전에 기준선을 설정 (빌드 성공 + 핵심 게임플레이/렌더링/네트워크 스모크 테스트)
- 각 변경을 작고 검증 가능한 단위로 유지하며, 여러 시스템에 걸친 리팩토링을 한 번에 수행하지 않기
- 다음 우선순위로 넘어가기 전에 각 단계에서 동작을 검증
- 외부/자동 생성 파일은 수정 금지 (`d3dx12.h`, `DDSTextureLoader12.*`, `Resource.h`, `targetver.h`)

## 점진적 실행 순서 (안전)
1. 현재 책임과 의존성을 매핑 (씬/렌더/네트워크/리소스 소유권)
2. 확인된 아키텍처와 알려진 코드 스멜에 대한 문서 작성/업데이트
3. 동작 보존 검증과 함께 한 번에 하나의 모듈씩 리팩토링
4. 모듈 경계가 안정된 후에만 디렉토리 재구성 수행
5. 구조적 변경이 확정된 후 성능 프로파일링 및 최적화 실행
